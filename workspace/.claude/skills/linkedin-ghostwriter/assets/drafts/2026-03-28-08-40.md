🧠 𝐂𝐨𝐧𝐭𝐞𝐱𝐭 𝐄𝐧𝐠𝐢𝐧𝐞𝐞𝐫𝐢𝐧𝐠 > 𝐏𝐫𝐨𝐦𝐩𝐭 𝐄𝐧𝐠𝐢𝐧𝐞𝐞𝐫𝐢𝐧𝐠

I spent two months writing better prompts. Longer system messages. More examples. More guardrails.

😅 The agent still forgot what project it was working on by turn 3.

Then I stopped prompting and started filing:

- `CLAUDE.md` — project rules, architecture, what not to touch
- `SOUL.md` — who the agent is, how it behaves
- `MEMORY.md` — what happened yesterday, decisions made, lessons learned

Three markdown files. Zero prompt engineering. The agent reads them at session start inside every #OpenHarness sandbox.

📌 The hard part: MEMORY.md goes stale. I'm still figuring out when to prune vs. append. If you've cracked that, I want to hear it.

```
git clone https://github.com/ryaneggz/open-harness.git
cat SOUL.md MEMORY.md CLAUDE.md
```
