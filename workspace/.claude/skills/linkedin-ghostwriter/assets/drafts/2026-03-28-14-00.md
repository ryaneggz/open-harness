🫣 𝐃𝐞𝐦𝐨: 𝐅𝐥𝐚𝐰𝐥𝐞𝐬𝐬. 𝐏𝐫𝐨𝐝𝐮𝐜𝐭𝐢𝐨𝐧: 𝐏𝐚𝐧𝐢𝐜.

My agent crushed 100 test invoices in the sandbox. Client sends their real export — 47K rows. Row 3,211: null in `unit_price`.

😅 Agent didn't crash. It silently skipped 800 rows with missing fields and reported "complete."

🧠 Sandbox testing proves your agent works on 𝘺𝘰𝘶𝘳 data. Client data has nulls where you expected numbers, dates in three formats, and SKUs that are someone's first name.

📌 What I fixed:
- Agent writes a `validation_report.md` before processing
- MEMORY.md logs every skipped row with a reason
- Test fixtures now use anonymized client exports, not my clean samples

The #OpenHarness sandbox caught the logic bugs. It didn't catch the data bugs.
