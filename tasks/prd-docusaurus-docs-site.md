# PRD: Docusaurus v3 Docs Site at `apps/docs` (oh.mifune.dev)

> Source spec: `.claude/specs/oh-mifune-docs-site.md`
> Tracking issue: [#164](https://github.com/ryaneggz/open-harness/issues/164)

## 1. Introduction / Overview

Scaffold a production-ready Docusaurus v3 TypeScript documentation site for Open Harness at `apps/docs/`. The site is the primary landing page at `oh.mifune.dev` and replaces ad-hoc Markdown rendering on GitHub. It is deployed to GitHub Pages from `main` via GitHub Actions.

Brand: **Mifune Open Harness**. Tagline: _Run Claude, Codex, Gemini, and Pi side-by-side from one Docker-powered agent harness._

## 2. Goals

- Docusaurus v3 (latest stable) TypeScript template, pnpm-workspace compatible
- Primary URL `https://oh.mifune.dev` (root `baseUrl: "/"`); GitHub Pages fallback documented
- `apps/docs/static/CNAME` contains `oh.mifune.dev`
- Initial docs tree per spec section "Initial Documentation Structure"
- Homepage explains Open Harness in under 10 seconds
- Quickstart gets a user from zero to `oh shell`
- One CLI page per major `oh` command (`onboard`, `sandbox`, `shell`, `clean`)
- Architecture page explains: one container, N worktrees, one daemon, multiple agents
- `pnpm docs:dev` / `docs:build` / `docs:serve` work from repo root
- GitHub Actions: build on PR, deploy on push to `main`
- Dark mode default; `trailingSlash: false`; MDX enabled; search-ready
- Existing root `docs/` preserved untouched

## 4. Functional Requirements

- FR-1: All site config goes in `apps/docs/docusaurus.config.ts` (TypeScript only)
- FR-2: Sidebars autogenerated via `apps/docs/sidebars.ts` for the initial pass
- FR-3: Docs route is `/docs`; homepage is `/` and is a custom React page (not `docs/intro.md` rendered as homepage)
- FR-4: Edit-this-page links must point at `https://github.com/ryaneggz/open-harness/edit/main/apps/docs/`
- FR-5: Dark mode is the default theme; user toggle still available
- FR-6: `static/CNAME` is committed and contains exactly the line `oh.mifune.dev`
- FR-7: GitHub Actions workflow uses official GitHub Pages actions (no third-party deploy actions)
- FR-8: Existing root `docs/` directory MUST NOT be deleted, moved, or modified
- FR-9: Root `package.json` adds three scripts: `docs:dev`, `docs:build`, `docs:serve`
- FR-10: pnpm workspace YAML includes `apps/docs`
- FR-11: All MD/MDX pages include valid frontmatter (`title`, `sidebar_position` where part of a sidebar group)
- FR-12: No emojis in committed docs (per project convention)

## 5. Non-Goals (Out of Scope)

- Migrating existing root `docs/*.md` files into `apps/docs/docs/` (deferred to follow-up PR per spec section "Migration Strategy")
- Search backend (Algolia DocSearch); spec says "search-ready" — local lunr fallback only via theme defaults; no Algolia keys required
- Versioned docs (single version only)
- i18n beyond `en`
- Custom React components beyond the homepage
- Replacing `README.md` install instructions
- Configuring Cloudflare DNS (the user owns `mifune.dev` and points the CNAME externally)
- Adding analytics, OG image generation, RSS, or blog plugin
- Migrating wiki content under `docs/wiki/`
- Auto-generated CLI reference from source — pages are hand-written

## 6. Design Considerations

- Brand name: **Mifune Open Harness** (header / titles); product short name: **Open Harness**
- Theme: dark default, accent color set via CSS custom properties in `src/css/custom.css`
- Hero copy: spec's core message verbatim
- Logo: simple SVG placeholder OK (a follow-up PR can replace it)

## 7. Technical Considerations

- Docusaurus v3 default Node ≥ 18; root `engines.node` already says `>= 20` ✓
- pnpm 10 is used at root → use `pnpm/action-setup@v4` with `version: 10` in CI
- Docusaurus v3 default depends on React 18; ensure no peer-conflicts with workspace's pnpm overrides
- `swizzle` not required for the initial pass
- `trailingSlash: false` means GitHub Pages should serve `/docs/intro` not `/docs/intro/`
- Custom domain via `static/CNAME` is canonical for GitHub Pages

## 8. Success Metrics

- New developer reads homepage and reaches `oh shell` in under 10 minutes
- `pnpm docs:build` finishes in < 60s on a clean install
- PR builds run in < 4 minutes end-to-end on CI
- Site loads at `https://oh.mifune.dev` after merge to `main` (DNS configured by user)
