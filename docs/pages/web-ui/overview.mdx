# Web UI

`@openharness/web-ui` is a vendored fork of [`@mariozechner/pi-web-ui`](https://github.com/badlogic/pi-mono), a library of Lit web components that deliver a complete AI chat interface in the browser — no backend required in Phase 1.

## What it is

The web UI is a static single-page application (SPA) built from reusable Lit components. It connects directly to LLM provider APIs from the browser, stores session history in IndexedDB, and lets users configure their own API keys through a settings dialog.

| Feature | Description |
|---------|-------------|
| Chat interface | Streaming message history with tool execution display |
| Multi-provider | Anthropic, OpenAI, Ollama, LM Studio, vLLM, and OpenAI-compatible APIs |
| Attachments | PDF, DOCX, XLSX, PPTX, images with preview and text extraction |
| Artifacts | Interactive HTML, SVG, Markdown with sandboxed execution |
| Storage | IndexedDB-backed sessions, API keys, and settings |
| CORS proxy | Automatic proxy handling for browser-to-API requests |

## Why we vendored it

The upstream package (`@mariozechner/pi-web-ui`) is tied to a specific version of `@mariozechner/pi-ai`. Open Harness pins its own version of `@openharness/web-ui` to control upgrades independently and to apply harness-specific patches without waiting on upstream releases.

The vendored copy lives at `packages/web-ui/` with the example app at `packages/web-ui/example/`.

## Deployment (Phase 1)

The example app is built into a static bundle and served by an nginx container defined in `.devcontainer/docker-compose.web-ui.yml`. The sandbox container and the web-ui container share the default compose network — the sandbox can reach the UI by service name (`web-ui`).

Users expose the UI publicly through the cloudflared tunnel they already operate. See [Setup](/open-harness/web-ui/setup) for details.
