# Quickstart

**Only host dependency:** [Docker](https://docs.docker.com/get-docker/).

## 1. Clone and configure

```bash
git clone https://github.com/ryaneggz/open-harness.git && cd open-harness
cp .devcontainer/.example.env .env        # configure name, password, etc.
```

See [Configuration](/open-harness/guide/configuration) for all available environment variables.

## 2. Start the sandbox

```bash
docker compose -f .devcontainer/docker-compose.yml up -d --build
```

## 3. Connect to the sandbox

Pick whichever method fits your setup:

### Option A — Terminal (works everywhere)

```bash
docker exec -it -u sandbox openharness bash   # use your SANDBOX_NAME
```

### Option B — VS Code Attach to Container (local)

Install the [Dev Containers](https://marketplace.visualstudio.com/items?itemName=ms-vscode-remote.remote-containers) extension, then:

`Cmd+Shift+P` → **"Attach to Running Container"** → select your sandbox.

### Option C — VS Code Remote SSH + Attach (remote server)

If Docker is running on a remote host, SSH into the host first, then attach to the container.

1. Add an entry to your local `~/.ssh/config` so credentials forward automatically:

   ```
   Host openharness
     HostName your-server-ip
     User openharness
     ForwardAgent yes
   ```

2. In VS Code: **Remote-SSH: Connect to Host** → `openharness`
3. Once connected to the remote host: **Attach to Running Container** → select your sandbox.

### Option D — SSH directly into sandboxes

Enable the `sshd` overlay and SSH straight into any sandbox on a unique port. This also enables sandbox-to-sandbox communication and CI/CD integration.

See the [Multi-Sandbox SSH guide](/open-harness/guide/multi-sandbox-ssh) for full setup.

## 4. Onboard (one-time)

Inside the sandbox, authenticate the tools you'll use:

```bash
gh auth login                    # GitHub CLI
gh auth setup-git                # git credential helper (no SSH keys needed)
pi                               # Pi Agent OAuth — powers Slack, heartbeats, extensions
```

See [Onboarding](/open-harness/getting-started/onboarding) for the full walkthrough.

## 5. Start working

```bash
claude                           # terminal coding agent
pi                               # automations — Slack, heartbeats, extensions
```

## Cleanup

```bash
docker compose -f .devcontainer/docker-compose.yml down -v    # containers + volumes
```

Omit `-v` to keep auth volumes (Claude Code, GitHub CLI, Cloudflare tokens) across rebuilds.
