# Configuration

All sandbox configuration is done through environment variables. Docker Compose reads a `.env` file automatically from the project root.

## Setup

Copy the example env file and edit to taste:

```bash
cp .devcontainer/.example.env .env
```

## Environment Variables

### Sandbox

| Variable | Default | Description |
|----------|---------|-------------|
| `SANDBOX_NAME` | `sandbox` | Container and project name |
| `SANDBOX_PASSWORD` | `changeme` | Sandbox user password (only used with [sshd overlay](/open-harness/guide/overlays#ssh-server-access)) |

### Timezone

| Variable | Default | Description |
|----------|---------|-------------|
| `TZ` | `America/Denver` | Container timezone |

### Heartbeats

| Variable | Default | Description |
|----------|---------|-------------|
| `HEARTBEAT_AGENT` | `claude` | Agent CLI for heartbeat tasks (`claude`, `codex`, `pi`) |
| `HEARTBEAT_ACTIVE_START` | _(empty)_ | Active window start (e.g. `08:00`) |
| `HEARTBEAT_ACTIVE_END` | _(empty)_ | Active window end (e.g. `18:00`) |

### SSH (overlay)

These only apply when the corresponding SSH overlay is enabled.

| Variable | Default | Description |
|----------|---------|-------------|
| `HOST_SSH_DIR` | `~/.ssh` | Host SSH directory to mount read-only (with `ssh.yml` overlay) |

### Slack bot (overlay)

These only apply when the `slack.yml` overlay is enabled.

| Variable | Default | Description |
|----------|---------|-------------|
| `SLACK_APP_TOKEN` | _(empty)_ | Slack app-level token (`xapp-...`) |
| `SLACK_BOT_TOKEN` | _(empty)_ | Slack bot token (`xoxb-...`) |
| `OPENAI_API_KEY` | _(empty)_ | OpenAI API key for LLM provider |

### Port forwarding

| Variable | Default | Description |
|----------|---------|-------------|
| `PORT` | `3000` | Host port for dev server (only relevant if you add a port mapping overlay) |

## Precedence

Docker Compose resolves variables in this order:

1. Shell environment (e.g. `SANDBOX_NAME=foo docker compose up`)
2. `.env` file in the project root
3. Default values in `docker-compose.yml` (the `:-` fallbacks)

Shell environment always wins, so you can override `.env` values on the command line.
