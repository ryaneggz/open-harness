# Releases

Open Harness uses CalVer (Calendar Versioning) for releases.

## Tag format

`YYYY.M.D` for the first release of the day (e.g., `2026.4.4`). For multiple releases on the same day, append an increment: `2026.4.4-2`, `2026.4.4-3`, etc.

## Creating a release

```bash
git tag 2026.4.4
git push origin 2026.4.4
```

This triggers the build workflow which builds and pushes:
- `ghcr.io/ryaneggz/open-harness:2026.4.4`
- `ghcr.io/ryaneggz/open-harness:latest`

## Using the /release skill

From inside the sandbox, the `/release` skill automates the full process:

```
/release
```

It creates a release branch, tags it, pushes to trigger CI, and waits for the build to complete.
